System Prompt Context Comparison

Reference notes comparing how Hermes Agent and FFT_nano handle Markdown context files in system messages, especially after starting a fresh session and across later turns.

Key distinction: the older FFT_nano prompt-cache work added stable/ephemeral machinery. The newer Hermes-inspired work changes what belongs in each layer: durable policy stays stable, while operational and high-churn context moves to the ephemeral suffix.

Terms

Term Meaning Good Fit
Stable Provider-cacheable prompt prefix expected to stay mostly identical across runs. Identity, safety, durable policy, IPC schemas, workspace paths.
Ephemeral Per-run suffix rebuilt for the current request/session turn. Current time, request metadata, active memory, TODOs, skill catalog, retrieved context.

Hermes Agent

Hermes builds a fresh system prompt snapshot for a new session. Later turns generally reuse that session-start snapshot rather than re-reading Markdown files every turn.

FFT_nano Before Hermes-Inspired Split

FFT_nano already had stable/ephemeral prompt layers, but too much behavior-shaping context lived in the provider-cacheable stable layer.

FFT_nano After Hermes-Inspired Split

Source New Layer Reason
Base identity/safety/tool truthfulness Stable Durable policy that should remain consistent across runs.
IPC schemas, workspace paths, completion expectations Stable Stable host contract and safety-critical operating rules.
SOUL.md Stable Identity, tone, safety, and durable user/policy context.
NANO.md Ephemeral Operational guidance can trigger tool use, TODO reads, or delegation.
MEMORY.md Ephemeral High-churn durable memory detail should be current per run, not cached as permanent policy.
TODOS.md Ephemeral Active mission/task state should update per run.
Canonical files Ephemeral Potentially important, but still content context rather than cacheable base policy.
Daily memory files Ephemeral Recent, high-churn context.
Skill catalog Ephemeral Capability hints are action-triggering and should not live in the cached prefix.
Request metadata, current time, request id, host overlay Ephemeral Per-run facts.

Practical Recommendation

Stable Should Be Boring

Keep identity, safety, truthfulness, destructive-command policy, IPC schemas, and stable workspace contracts in the cacheable prefix.

Ephemeral Should Be Current

Keep skills, TODOs, memory, operational guidance, retrieved context, and active runtime metadata in the per-run suffix.
