Architecture

FFT_nano is a single Node.js process that receives messages from Telegram/WhatsApp, stores data in SQLite, runs the container, and sends agent in an isolated responses back.

Memory System

  • Per-group: groups/<group>/SOUL.md + memory/*.md
  • Global: groups/global/SOUL.md (read by all, write from main only)
  • Session: Each group maintains a conversation session persisted by Pi
üìù What Goes in Memory
Your agent remembers conversations, preferences, and context. If you discuss sensitive information (finances, personal details), that stays in your local groups/ folders. Nothing goes to external servers except via your LLM provider. Your agent can help you manage or clear memory if needed.

Scheduling

Users can schedule recurring or one-time tasks. Tasks run as full agents in their group's context.

  • Cron expressions ‚Äî e.g., 0 6 * * * (daily at 6am)
  • Intervals ‚Äî e.g., every 30 minutes
  • One-time ‚Äî ISO timestamp

Task runs are logged to SQLite with duration and result.

‚è∞ Scheduling Awareness
Each scheduled task runs your agent and uses API calls (which may cost money). If you schedule tasks every 5 minutes, that adds up. Start with less frequent schedules and adjust based on your needs.

Container Isolation

Agents run inside Apple Container (macOS) or Docker (Linux). Each invocation spawns a container with mounted directories.

üõ°Ô∏è What This Means for You
Your agent runs in an isolated sandbox. It can only see files you explicitly give it access to. Even if something goes wrong, your main computer stays protected. The agent is the farmhand‚Äîbut it's in a separate building, not in your house.
üë®‚Äçüåæ Farmer Note:
This is designed so you can let your agent work autonomously without worrying it'll accidentally delete your photos or mess up your system. It can only touch what you tell it to.

Contributing

FFT_nano is a product-focused fork. Contributions should:

  • Preserve the security model (container isolation + minimal mounts)
  • Stay understandable (prefer direct code over framework layers)
  • Include threat-model notes for new integrations

Accepted: bug fixes, security fixes, farm-assistant features, docs updates.

Avoid: "platform/framework" work that adds abstraction without user value.
