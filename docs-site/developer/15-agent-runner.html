Pi Runner And Coding Worker

Canonical source: docs-site/developer/15-agent-runner.md.

Current execution split:

  • src/container-runtime.ts chooses docker vs host.
  • src/pi-runner.ts launches pi, builds the prompt, streams tool events, and parses output.
  • src/coding-orchestrator.ts manages real coding worker runs, isolated worktrees, and structured results.

Important runtime controls:

  • toolMode=read_only|full controls plan vs execute tool access.
  • workspaceDirOverride lets the host run pi inside an isolated coder worktree.
  • toolExecutions are returned so the host can report commands and tests.
