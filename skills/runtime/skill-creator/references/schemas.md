# Evaluation Schemas

This document defines the JSON structures used throughout the skill evaluation system.

## evals.json

The main eval configuration file. Each eval has a unique `id` and contains the prompt, expected output description, and optional assertions.

```json
{
  "skill_name": "example-skill",
  "evals": [
    {
      "id": 1,
      "prompt": "The user's task prompt",
      "expected_output": "Description of expected result",
      "files": ["path/to/input/file.csv"],
      "assertions": [
        {
          "name": "Output file exists",
          "type": "file_exists",
          "path": "output/result.csv"
        },
        {
          "name": "Contains required columns",
          "type": "file_contains",
          "path": "output/result.csv",
          "content": ["date", "revenue", "profit_margin"]
        }
      ]
    }
  ]
}
```

### Assertion Types

| Type | Fields | Description |
|------|--------|-------------|
| `file_exists` | `path` | Checks that a file was created |
| `file_contains` | `path`, `content` (array of strings) | Checks that file contains all specified strings |
| `file_not_contains` | `path`, `content` (array of strings) | Checks that file does not contain specified strings |
| `output_contains` | `content` (array of strings) | Checks stdout/output contains all specified strings |
| `output_not_contains` | `content` (array of strings) | Checks stdout/output does not contain specified strings |
| `json_schema` | `path`, `schema` (object) | Validates JSON file against a schema |
| `custom` | `check` (string) | Freeform description for manual grading |

## grading.json

Saved per-run. Contains the results of evaluating each assertion.

```json
{
  "eval_id": 0,
  "eval_name": "descriptive-name",
  "configuration": "with_skill",
  "timestamp": "2024-01-15T10:30:00Z",
  "expectations": [
    {
      "text": "Output file exists",
      "passed": true,
      "evidence": "Found output/result.csv (1.2KB)"
    },
    {
      "text": "Contains required columns",
      "passed": false,
      "evidence": "Missing: profit_margin. Found: date, revenue"
    }
  ]
}
```

**Important**: The `expectations` array must use the fields `text`, `passed`, and `evidence` — the eval viewer depends on these exact field names.

## eval_metadata.json

Saved per-eval. Contains the prompt and assertion definitions for a specific eval.

```json
{
  "eval_id": 0,
  "eval_name": "descriptive-name",
  "prompt": "The user's task prompt",
  "assertions": [
    {
      "name": "Output file exists",
      "type": "file_exists",
      "path": "output/result.csv"
    }
  ]
}
```

## benchmark.json

Aggregated results across all evals for an iteration. Used by the eval viewer.

```json
{
  "skill_name": "my-skill",
  "iteration": 1,
  "configurations": [
    {
      "name": "with_skill",
      "evals": [
        {
          "eval_id": 0,
          "eval_name": "descriptive-name",
          "prompt": "The user's task prompt",
          "assertions_passed": 2,
          "assertions_total": 3,
          "pass_rate": 0.667,
          "total_tokens": 84852,
          "duration_ms": 23332
        }
      ],
      "aggregate": {
        "mean_pass_rate": 0.833,
        "stddev_pass_rate": 0.117,
        "mean_tokens": 72145,
        "stddev_tokens": 12340,
        "mean_duration_ms": 21500,
        "stddev_duration_ms": 3200
      }
    },
    {
      "name": "without_skill",
      "evals": [
        {
          "eval_id": 0,
          "eval_name": "descriptive-name",
          "prompt": "The user's task prompt",
          "assertions_passed": 0,
          "assertions_total": 3,
          "pass_rate": 0.0,
          "total_tokens": 65432,

        "mean_tokens": 68000,
        "stddev_tokens": 8900,

    "pass_rate": 0.667,
    "total_tokens": 4145,
    "duration_ms": 2300
  }
}
```

### Ordering

Each with_skill configuration must appear before its baseline counterpart. For example:
- `with_skill` before `without_skill`
- `new_skill` before `old_skill`

## timing.json

Saved per-run. Contains token usage and timing data captured from the subagent notification.

```json
{
  "total_tokens": 84852,
  "duration_ms": 23332,
  "total_duration_seconds": 23.3
}
```

## feedback.json

Generated by the eval viewer. Contains user reviews for each run.

```json
{
  "reviews": [
    {
      "run_id": "eval-0-with_skill",
      "feedback": "the chart is missing axis labels",
      "timestamp": "2024-01-15T11:00:00Z"
    },
    {
      "run_id": "eval-1-with_skill",
      "feedback": "",
      "timestamp": "2024-01-15T11:01:00Z"
    }
  ],
  "status": "complete"
}
```

Empty `feedback` means the user approved the output.
